Release 37

What's New

CMemoryFile - This provides a simple to use interface to memory mapped files.

CPhysicalDiskFile - This class makes it easy to read data from a disk in raw format. In other words, this class reads a disk like it was a file without regard to file system on that disk. Want to read an individual sector on the disk? This is the class for you.

Bug Fixes/Enhancements

CEventLog - I added methods to create custom logs and enumerate through existing logs.

CQueue - Fixed a problem where, on multiple CPU systems, the add index would sometimes cause an exception. This has been fixed. It has also been rewritten for speed optimization. Adding to the queue does not prevent another thread from reading from the queue.

CExtensibleMarkupLanguageDocument - Christopher Remington (chrisr@silversoft.net) Found a bug where I wasn't initializing the m_ParseErrorEncountered member correctly. It is fixed now.

CExtensibleMarkupLanguageElement - Mats Johnsson (mats.johnsson@comfact.com) Found a couple of problems. One was AddAttribute() was not normalizing attribute values according to section 3.3.3 of the XML specification. He also added a new method called GetAttributeByName() the returns a pointer to the CExtensibleMarkupLanguageAttribute. Mats also suggested a method called GetParent() that accepts a name of an element and will return the pointer to that element in the parentage of this element.

CExtensibleMarkupLanguageElement - Pei-Chiang Ma (north@uni-scape.com) (north@uni-scape.com) Found a bug in the section of code responsible for writing CDATA sections.

CFileTime - David LeBlanc (whisper@accessone.com) found a bug in the dynamic_cast operator in Visual C++ 4.2.

CSerialFile - Glenn Brude (d40480@algonet.se) found a bug in Write(). If everything didn't get sent in the first call, it would not loop. If you were sending lots of bytes, not all of them may have been sent. DOH!

CSerialFile - Jim Peng (jimp@datalink.net) found a minor bug in Purge(). I was returning a value from GetLastError() instead of FALSE.
