Release 33

In summary, I've fixed some design problems with XML document creation and discovered where Microsoft screwed me again in the networking API.

I'm happy with the features of this release so I will now turn my attention to UNICODE problems. Look for Release 34 to address UNICODE issues.

What's New

CXMLArchive
This class is modeled after the CArchive MFC class. It allows you to serialize (or persist) objects to and from XML documents.
wfc_parse_iso_8601_string
This function parses ISO8601 dates into a CTime or COleDateTime object.
XMLUser Sample
This sample program shows you how to use the XML classes to create a document. It dumps the user database from one (or more) machines to an XML file. It shows how to use the CXMLArchive class.

Bug Fixes/Enhancements

CExtensibleMarkupLanguageDocument - Reviewed for const correctness.

CExtensibleMarkupLanguageElement - Made it a lot easier (and possible) to create XML documents. I also reviewed it for const correctness. I got so caught up in trying to get Release 32 out the door, I released a brain-dead object. You could parse XML documents but you couldn't create them. Pretty silly.

CNetwork - Added a new protected data memeber because Microsoft changed (again) the personality of a networking API.

CSimpleSocket - Thanks go to zschen (zschen@ms.chttl.com.tw) for founding a bug in the Write( CString ) method where I wasn't sending the trailing NULL. It is fixed now.
